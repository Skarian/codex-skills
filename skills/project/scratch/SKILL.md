# Scratch

Create a Markdown scratch file in the repo root and use it as the working surface before editing the final target file.

Name the file from the task when obvious, such as `readme-scratch.md`; otherwise use `scratch.md`.

Keep the scratch file visible at the repo root, show the user the path, and do not replace the final target until the user asks.
